# v0.2.0

## Capability Gap Closure

- Added deterministic structured intent facets for task type, evidence needs, write target, mutation kind, destructive intent, and ambiguity diagnostics.
- Added mission acceptance checks that record pass/fail/needs-more-work and force another step only for actionable missing proof such as write receipts, explicit web evidence, word-count checks, or explicit vault-before-answering requirements.
- Added resumable mission-ledger state for acceptance status, remaining actions, resume count, and last safe step.
- Added retrieval coverage metadata for vault-context and semantic-search results so Run Details can show exact, sampled, indexed, or fallback evidence confidence.
- Expanded research memory with duplicate detection, review, compact-with-backup, and trash-with-backup tools.
- Added one-shot browser approval payloads for reversible high-risk actions while keeping credential, payment, purchase, upload, and executable/script actions blocked.
- Tightened reflex completion recommendations so the agent suggests available recovery tools when evidence is missing without broadening the authority layer.
- Added unit and Playwright coverage for acceptance retry, structured intent, coverage scoring, browser approval, and research-memory hygiene.

## Versioning

- Bumped `package.json`, `package-lock.json`, and `manifest.json` to `0.2.0`.
- Added `versions.json` for Obsidian-compatible version metadata.
