Daily-use capability

Start with the task, not the settings.

Chat is the mission console. The agent chooses the relevant capability and only asks for setup when a required connection is missing.

01

Vault research

Read the current note, search the vault and web, compare evidence, then append sourced markdown.

notes + web → cited brief
02

Code work

Create Python, TypeScript, JavaScript, C/C++, HTML/CSS, Rust, Go, Java, or C# in trusted workspaces with validation and receipts.

task → patch → proof
03

Linear

Turn research or a note into a structured work item through fixed, permission-aware actions.

brief → issue → readback
04

GitHub

Prepare commits and draft pull requests, inspect reviews, and keep merge separately approved.

change → PR → receipt

Designed for trust

Agentic where useful.
Explicit where consequential.

01

Native and local-first

The assistant lives in Obsidian’s right sidebar and works through vault-relative paths and Obsidian APIs.

02

Append first

Ordinary writing appends safely. Replacement and destructive actions require intent, backup, or approval.

03

Visible proof

Run Details keeps plans, tool activity, validation, and write receipts available without crowding the chat.

04

Minimal setup

One working mode and connection-derived readiness replace a wall of independent capability toggles.
